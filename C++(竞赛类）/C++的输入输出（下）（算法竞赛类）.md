# 一、OJ 题目输入情况汇总

在竞赛的 OJ 题目中，输入情况大概可以分为单组测试用例和多组测试用例。

## 1. 单组测试用例

程序运行 1 次，只处理一组数据。这种情况最简单，直接使用 `cin` 读取即可：

```cpp
int a, b, c;
cin >> a >> b >> c; 
```

## 2. 多组测试用例

程序运行 1 次，会处理多组数据。这又分为三种常见情况：

* **测试数据组数已知**
题目会先给我们一个数字代表测试组数或者询问次数 `q`。当有 `q` 次询问的时候，我们常用 `while(q--)` 来处理，这是非常方便的方式：
```cpp
int t, a, b;
cin >> t;
while(t--) // t次循环非常方便
{ 
    cin >> a >> b;
    cout << a + b << '\n';
}
```


* **测试数据组数未知**
题目直接给一堆数据，直到输入结束。在 C++ 中，流对象 `cin` 可以被用作布尔值来检查流的状态。如果成功读取到值，流对象将被转换为 `true`，循环继续；如果读取失败（如遇到输入结束符），布尔值为 `false`，循环停止：
```cpp
int a, b;
while(cin >> a >> b) // 读取失败时循环停止
 {
    cout << a + b << '\n';
}
```

* **特殊值结束测试数据**
例如，当输入特定条件结束时，可以巧妙利用**逗号表达式**处理：
```cpp
int a, b;
while(cin >> a >> b, a && b)// a和b不全为0时继续
 { 
    cout << a + b << '\n'; 
}
```

# 二、输入中的特殊技巧

在做题时，选对数据读取方式能省下大把时间！

* **含空格字符串的特殊处理**
遇到带空格的句子，不一定非要完整的读取整个带空格的字符串。把它们当成空格隔开的每一个字符串，**当做一个单词处理更方便**。直接用 `while(cin >> s)` 逐个读取，不仅写起来快，也避免了处理带空格字符串的各种换行符残留问题 。


* **数字的特殊处理方式**
控制台输入的 123 本质上是一个字符序列。程序可能将其解析成整型 `cin >> num`，也可能解析成字符串 `cin >> s` 。


> **💡 技巧：**
> 比如题目要求你对数字的“每一位”进行操作时（例如判断每一位数字与对应奇偶性是否一致），**直接当字符串读取**！然后用 `s[i]` 遍历，最后再用 `stoi(s)` 转换成数字输出即可。这比用 `%10` 和 `/10` 取位要清爽得多 。

# 三、scanf/printf 与 cin/cout 的区别及关流代码

我们在刚接触C++时不了解他们的区别和用谁，让我们通过下面分析来了解他们的区别及优化吧

## 1. 格式控制差异

* **cin/cout**：会根据变量类型自动处理，避免格式化错误，更加易用 。此外，`cout` 默认不会输出六位小数，自动忽略小数点后多余的0 。


* **scanf/printf**：不能自动识别类型，需要手动指定格式。但它**格式化输出更精确直观**，特别适合复杂格式（`printf` 打印浮点数时默认保留 6 位） 。



## 2. 性能差异与优化（重点！）

通常情况下，`scanf/printf` 比 `cin/cout` 快 。这是因为 C++ 标准库默认会将 C++ 流对象与 C 标准库流对象**同步**，且 `cin` 和 `cout` 之间存在**绑定**关系，导致性能下降 。

> **🚀 优化秘籍：**
> 当输入输出数据量较大时，使用普通的 `cin/cout` 经常会遇到 **Time Limit Exceeded (TLE)**。只要在 `main` 函数开头加上这两行代码，就能取消同步与绑定，速度大幅提升：
> ```cpp
> ios::sync_with_stdio(false); // 取消与C语言输入输出缓冲区的同步
> cin.tie(0); // 取消cin和cout的绑定
> ```

> **🔥 核心避坑指南：**
> 在使用上述“关流”代码优化后，输出换行时**一定要用 `'\n'` 替代 `endl**`！
> 因为 `endl` 的底层逻辑不仅会换行，还会**强制刷新输出缓冲区**。我们刚刚才为了提速关掉了各种同步，如果每输出一行就用 `endl` 强行刷新一次，会极大拖慢输出速度，让我们的优化功亏一篑！

## 📌 总结建议

* **数据量较小**（10^6 以内）：用 `cin/cout` 或 `scanf/printf` 凭习惯随便选 。


* **数据量较大**（10^9 左右）：更推荐使用 `scanf/printf`，或者**一定要加上关流优化代码并且使用 `'\n'` 的 `cin/cout` 。
